# 접근성 (Accessibility, a11y)

> 토스 frontend-fundamentals 기반. 출처: https://github.com/toss/frontend-fundamentals

**원칙**: UI가 보조기술(스크린리더)·키보드·터치로도 해석·조작 가능해야 한다. **모바일 공유가 핵심인 서비스**라면 터치 타깃·이미지 대체텍스트·시맨틱 구조가 특히 중요하다.

## 1. 시맨틱 태그 우선

**Before** — 클릭만 되는 `div`. 키보드·스크린리더 불가.

```tsx
<div onClick={share}>공유하기</div>
```

**After**

```tsx
<button type='button' onClick={share}>
  공유하기
</button>
```

`div`/`span`에 `onClick`을 붙여야만 한다면 `role`, `tabIndex={0}`, `onKeyDown`(Enter/Space)을 함께 제공한다. 가능하면 `button`/`a`를 쓴다.

## 2. 이미지 대체 텍스트

```tsx
// Before
<Image src={photo.url} width={400} height={300} />

// After — 의미 있는 alt (장식용이면 alt="")
<Image src={photo.url} alt={`${authorName}님의 프로필 사진`} width={400} height={300} />
```

## 3. 키보드 포커스 & 모달

- 모달(shadcn `Dialog`)은 열릴 때 포커스 트랩, 닫힐 때 트리거로 포커스 복귀가 되어야 한다(shadcn 기본 제공).
- 커스텀 닫기 버튼을 추가할 때 `aria-label="닫기"`를 붙인다.
- 주의: shadcn `DialogContent`는 기본 X 버튼을 렌더하므로, 커스텀 X를 둘 땐 `showCloseButton={false}`로 둔다 — 중복 버튼은 포커스 순서도 혼란시킨다.

## 4. 모바일 터치 타깃

- 터치 가능한 요소는 최소 44×44px 권장. 작은 아이콘 버튼은 `padding`으로 히트 영역을 키운다.

## 5. 폼 라벨 연결

```tsx
<label htmlFor="title">제목</label>
<input id="title" ... />
```

shadcn `FormField`/`FormLabel`은 라벨-컨트롤 연결을 자동 처리하므로 그대로 활용한다.

## smell → remedy

| Smell              | Remedy                                          |
| ------------------ | ----------------------------------------------- |
| 클릭만 되는 `div`  | `button`/`a` 또는 `role`+`tabIndex`+`onKeyDown` |
| `alt` 누락         | 의미 있는 alt(장식은 `alt=""`)                  |
| 라벨 없는 input    | `htmlFor`/`id` 연결, `FormLabel` 사용           |
| 색만으로 상태 전달 | 텍스트/아이콘 보조                              |
| 작은 터치 타깃     | 최소 44px 히트 영역                             |

## 심화 감사

페이지 단위의 종합 UX/접근성 감사는 `web-design-guidelines` 스킬(외부 가이드 동적 fetch → `file:line` 리뷰)을 활용한다.

## 과설계 경고

- 모든 요소에 `aria-*`를 남발하면 오히려 스크린리더가 시끄러워진다. 시맨틱 태그로 충분한 경우 `aria`를 추가하지 않는다.
